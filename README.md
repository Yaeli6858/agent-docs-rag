# Agentic-Docs RAG Assistant 🤖

An advanced RAG (Retrieval-Augmented Generation) system designed to bridge the gap between AI coding agents and human developers. This application aggregates, indexes, and queries technical documentation generated by agents like **Cursor, Claude Code, and Kiro**, providing a unified knowledge layer for technical decision-making.

## 🎯 Project Overview
As AI coding agents become integral to the development workflow, they generate a vast amount of metadata (instructions, specs, plans) stored in fragmented `.md` files. This project provides a specialized RAG pipeline to:
* **Centralize Knowledge:** Consolidate documentation from multiple AI agents.
* **Enable Semantic Search:** Find technical decisions based on meaning, not just keywords.
* **Enhance Onboarding:** Help new developers understand "why" certain technical paths were taken.

## 🛠 Tech Stack
* **Framework:** [LlamaIndex](https://www.llamaindex.ai/) (Data framework for LLM applications)
* **Vector Database:** [Pinecone](https://www.pinecone.io/) (High-performance vector storage)
* **Embeddings:** [Cohere](https://cohere.com/) (`embed-multilingual-v3.0`)
* **LLM:** [Groq](https://groq.com/) (Running Llama 3 for ultra-fast inference)
* **UI:** [Gradio](https://gradio.app/) (For an interactive chat interface)

## 🏗 System Architecture

### 1. Data Ingestion Pipeline (The "Factory")
The system processes raw documentation through a modular multi-step pipeline:
* **Loading:** Recursively reads `.md` files from agent-specific directories using `SimpleDirectoryReader`.
* **Chunking:** Utilizes `SentenceSplitter` with an optimized chunk size (512 tokens) and overlap to maintain context.
* **Embedding:** Converts text into high-dimensional vectors using Cohere's multilingual model.
* **Indexing:** Synchronizes vectors and metadata (source file, tool name, etc.) to a Pinecone vector index.



### 2. Query & Inference (The "Assistant")
The inference layer is designed for precision and reliability:
* **Semantic Retrieval:** Fetches the top-K relevant nodes using vector similarity search.
* **Post-Processing:** Applies a `SimilarityPostprocessor` with a 0.7 threshold to filter out irrelevant noise.
* **Response Synthesis:** Uses a custom-engineered System Prompt to ensure the LLM remains grounded in the provided context and avoids hallucinations.
