# Agentic-Docs RAG Assistant 🤖

An advanced Hybrid RAG (Retrieval-Augmented Generation) system designed to bridge the gap between AI coding agents and human developers. This application aggregates, indexes, and queries technical documentation generated by agents like **Cursor**, **Claude Code**, and **Kiro**, providing a unified knowledge layer for technical decision-making.

---

## 🎯 Project Overview

As AI coding agents become integral to the development workflow, they generate a vast amount of metadata (instructions, specs, plans) stored in fragmented `.md` files. This project provides a specialized RAG pipeline to:
* **Centralize Knowledge:** Consolidate documentation from multiple AI agents into one "Source of Truth".
* **Hybrid Search:** Combine Semantic Search (Pinecone) with Structured Data Extraction (JSON) for maximum precision.
* **Technical Governance:** Automatically extract and track coding rules, technical decisions, and system warnings.
* **Intelligent Routing:** Use an AI Router to decide whether to fetch data from a vector database or a structured schema.

## 🏗 System Architecture

The system is built on a modular, **Event-Driven Architecture** using LlamaIndex Workflows.

### 1. Data Ingestion & Extraction (The "Factory")
* **Vector Pipeline:** Loads `.md` files, chunks them using `SentenceSplitter`, and indexes them in **Pinecone** for semantic search.
* **Structured Extraction:** A dedicated engine (`extractor_engine.py`) uses a Pydantic schema to extract:
    * **Decisions:** Tech stack choices and architectural paths.
    * **Rules:** Coding standards and mandatory instructions.
    * **Warnings:** Technical debt, security risks, or "low-level" cautions.
* **Batch Processing:** Scans agent directories and generates a unified `project_knowledge.json`.

### 2. Intelligent Query Engine (The "Brain")

* **Smart Router:** Analyzes user intent to choose the best path:
    * **Structured Path:** Best for lists of rules or specific technical decisions.
    * **Semantic Path:** Best for general "How-to" or conceptual questions.
* **Event-Driven Workflow:** A multi-step process (`Retrieve` -> `Validate` -> `Synthesize`) that ensures responses are grounded and relevant.
