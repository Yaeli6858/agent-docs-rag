# 🤖 Agentic-Docs RAG Assistant

> A production-grade **Hybrid RAG system** that aggregates, indexes, and intelligently queries technical documentation generated by AI coding agents (Cursor, GitHub Copilot, Claude Code).

---

## 🎯 Project Overview

AI coding agents like **Cursor**, **GitHub Copilot**, and **Claude Code** continuously generate metadata — instructions, specs, plans, decisions — stored in fragmented `.md` files. Developers rarely read these files, yet they contain critical knowledge about the project.

This system solves that problem by providing a **unified knowledge layer** over all agent documentation, enabling developers to ask natural language questions and get precise answers.

### What makes this system special?

| Problem | Solution |
|---------|----------|
| Rules scattered across many files | Structured extraction into one JSON |
| Semantic search misses "list all" queries | Hybrid routing: JSON + Vector search |
| No validation of retrieved context | Event-Driven Workflow with confidence scoring |
| Multiple AI agents, multiple formats | Single unified pipeline for all agents |

## 🏗 System Architecture

The system is built across **3 phases**, each adding a layer of intelligence.

### Phase A — Semantic Pipeline (MVP)

A 5-step data pipeline that loads, processes, and indexes all `.md` files into Pinecone for semantic search.

```
Step 1: Load .md files (SimpleDirectoryReader)
    ↓
Step 2: Chunk into Nodes (SentenceSplitter: 512 tokens, 20 overlap)
    ↓
Step 3: Generate Embeddings (Cohere embed-multilingual-v3.0)
    ↓
Step 4: Index into VectorStoreIndex
    ↓
Step 5: Store & verify in Pinecone
```

### Phase B — Event-Driven Workflow

Refactored the query engine into a structured **LlamaIndex Workflow** with 3 steps, each communicating via typed Events:

```
User Query → StartEvent
    ↓
[Step: setup] — Retrieve top-3 nodes from Pinecone
    ↓ RetrievalEvent
[Step: validate] — Check if nodes exist & score ≥ 0.35
    ↓ ValidationEvent (or StopEvent if invalid)
[Step: generate] — Build prompt with context → LLM → Answer
    ↓ StopEvent → Final Answer
```

### Phase C — Structured Extraction + Smart Router

Added a parallel knowledge layer for list-based and rule-based queries:

```
All .md files
    ↓
[batch_processor] — Iterates over every file
    ↓
[extractor_engine] — LLM extracts structured items
    ↓
project_knowledge.json (decisions, rules, warnings)
```

Combined with a **Smart Router** that decides which path to use:

```
User Query
    ↓
[Smart Router] — LLM classifies query intent
    ↙               ↘
"structured"      "semantic"
    ↓                  ↓
JSON lookup       RAG Workflow
(all items)      (Pinecone search)
    ↘               ↙
      Final Answer
```

## 🚀 Getting Started

### Prerequisites

- Python 3.10+
- API keys for **Cohere** and **Pinecone**
- A Pinecone index named `code-docs-index` with **1024 dimensions**

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Yaeli6858/agent-docs-rag.git
cd agent-docs-rag

# 2. Install dependencies
pip install -r requirements.txt

# 3. Create .env file
```

Create a `.env` file in the root directory:

```env
COHERE_API_KEY=your_cohere_api_key
PINECONE_API_KEY=your_pinecone_api_key
```

## ▶️ Running the Project

### Step 1 — Index Documentation (Run once)

Loads all `.md` files, generates embeddings, and uploads to Pinecone:

```bash
python run_pipeline.py
```

Expected output:
```
=== Starting RAG Pipeline ===
--- Step 1: Loaded 10 documents successfully ---
--- Step 2: Created 87 nodes ---
--- Step 3: Model is ready ---
--- Step 4: Uploading 87 nodes to Cloud ---
✅ Verification successful!
📊 Total vectors in index 'code-docs-index': 87
=== ✅ Pipeline Finished Successfully ===
```

### Step 2 — Extract Structured Knowledge (Run once)

Extracts decisions, rules, and warnings into `project_knowledge.json`:

```bash
python extraction/batch_processor.py
```

Expected output:
```
--- Starting Batch Process ---
Found 4 files to process.
✅ Finished processing: general-copilot-instructions.md
✅ Finished processing: controller-instruction.md
✨ SUCCESS! Created database file: project_knowledge.json
```

### Step 3 — Launch Chat UI

Start the hybrid RAG assistant:

```bash
python app/run_workflow_chat.py
```

Then open your browser at `http://localhost:7860`

## 🧠 Key Design Decisions

### Why Event-Driven Workflow?
Breaking the query pipeline into discrete steps (`Retrieve → Validate → Generate`) makes each step independently testable and allows early termination when context quality is poor — preventing hallucinations.

### Why a Confidence Score Threshold (0.35)?
If Pinecone returns results with similarity score below 0.35, the retrieved context is not relevant to the question. Sending irrelevant context to the LLM causes hallucinations. The workflow stops early and returns an honest "I don't know."

### Why Hybrid Search (JSON + Vector)?
Pure semantic search returns only the **top-k** most similar chunks — it cannot guarantee returning **all** items in a list. The structured JSON layer enables complete, reliable retrieval for list-based queries (e.g., "give me ALL the coding rules").

### Why `embed-multilingual-v3.0`?
The documentation files contain mixed Hebrew and English content. This model handles both languages in the same embedding space, ensuring accurate semantic matching regardless of language.

## 🔍 Reflection

**What the agent handles well:**
- Specific "how-to" questions grounded in the documentation
- Complete lists of rules, decisions, and warnings
- Cross-file knowledge aggregation

**Current limitations:**
- Time-based queries ("what changed last week?") — no timestamp tracking
- The Router occasionally misclassifies ambiguous queries
- Structured extraction covers only the `/instructions` folder, not all agent directories

**What I would add next:**
- File change detection for automatic re-indexing
- Timestamp metadata for time-based filtering
- Confidence score display in the UI
- Support for more agent tools (Kiro, Claude Code directories)
